Connect GitHub to Your Domain

Complete guide to linking your GitHub project with your custom domain

1Prepare Your GitHub Repository

Ensure your project is properly set up on GitHub:

  1. Your code is pushed to GitHub
  2. Your repository has an index.html file
  3. Your project structure is organized
  4. All file paths use relative URLs
Use relative paths (e.g., "css/style.css") instead of absolute paths (e.g., "/css/style.css") for all your assets.

2Enable GitHub Pages

Turn on GitHub Pages for your repository:

  1. Go to your repository on GitHub
  2. Click on "Settings"
  3. Select "Pages" from the left sidebar
  4. Under "Source", select your branch (usually main or master)
  5. Select the folder (usually /root)
  6. Click "Save"

Note

GitHub will provide you with a URL where your site is published (username.github.io/repository).

4DNS Configuration

Configure your DNS records to point to GitHub:

# For apex domain (nagabu.art) Type: A Name: @ Value: 185.199.108.153 TTL: Automatic # Also add these other IP addresses for redundancy: 185.199.109.153 185.199.110.153 185.199.111.153 # For www subdomain (www.nagabu.art) Type: CNAME Name: www Value: yourusername.github.io TTL: Automatic
Always verify the current IP addresses for GitHub Pages in the official documentation as they may change.

5Create CNAME File

Add a CNAME file to your repository:

# Contents of CNAME file (no http:// or https://) nagabu.art

This file tells GitHub Pages which custom domain to use for your site.

Important

The CNAME file should be placed in your repository's root directory or in the same branch you selected for GitHub Pages.
